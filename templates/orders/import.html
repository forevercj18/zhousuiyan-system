兼容规则

1. 结构化导入：客户昵称、手机号码、地址、款式、客订来源、租金、预收押金、预定时间、发货时间、状态、发货单号。
2. “款式”列现在必须精确匹配产品管理中的产品名称；不存在的产品会直接报错，不会导入。
3. 兼容保留：租金渠道、押金渠道、经手人、操作时间及其它未识别列，会自动拼进订单备注，不会丢。
4. 同一份表里如果混有预定单，会优先按“订单类型/单据类型”分流；如果没有该列，则按状态自动识别预定单与正式订单。
5. 自动识别时，`待补信息 / 待确认 / 待跟进 / 待联系 / 可转正式订单 / 可转单 / 待转单 / 已转订单 / 已取消 / 已退款` 会按预定单导入；`待处理 / 待发货 / 已发货 / 使用中 / 已归还 / 已完成 / 已取消` 会按正式订单导入。
6. 识别为预定单的行会创建到预定管理，并把“预收押金”作为订金写入。
7. 建议流程：先点“预检查”看匹配结果和错误行，再点“确认导入”。
