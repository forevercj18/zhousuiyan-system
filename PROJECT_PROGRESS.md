# 项目进度文档（持续更新）

更新时间：2026-02-28
分支：`main`

## 1. 当前完成情况（可用）

### 1.1 订单中心与状态流
- 订单主流程可跑通：待处理 -> 待发货 -> 已发货 -> 已归还/已完成。
- 订单金额口径已统一为“租金”，押金单独记录，不再计入订单金额展示。
- 订单列表增加“转寄挂靠”列，且已修复重复展示问题：
  - 仅显示有效挂靠（`locked/consumed`）；
  - 同来源单号做数量合并后展示。

### 1.2 转寄候选与任务拆分
- 转寄中心已拆分两类操作：
  1. `重新推荐`：仅重算并更新挂靠，不生成任务；
  2. `生成任务`：按当前挂靠生成转寄任务（支持单条/多选）。
- 生成任务前会先校验“推荐来源 vs 当前挂靠”：
  - 若不一致，先切换挂靠到推荐来源，再生成任务。
- 任务区已做分栏：
  - `待操作`（默认）/`已完成`/`已取消`。

### 1.3 转寄规则（当前生效）
- 候选来源状态：`pending` / `confirmed` / `delivered`。
- 时间差规则：来源预定日期必须早于目标预定日期 **6 天及以上**（即严格 > 5 天）。
- 候选排序：先日期匹配，再距离优先（本地距离算法，无外部 API）。
- 重新推荐限制：若该 `order+sku` 已有 `pending` 转寄任务，则不可重推（前后端双重限制）。

### 1.4 仓库发货配置
- 系统设置页新增并已接通：
  - `warehouse_sender_name`
  - `warehouse_sender_phone`
  - `warehouse_sender_address`
- 当候选/推荐为仓库发货时，页面展示仓库发货人信息。

### 1.5 审计日志（可追溯）
- 转寄中心关键操作已落审计日志（模块=`转寄`）：
  - 重新推荐挂靠；
  - 生成任务（含“生成前自动切换挂靠”）；
  - 手动创建任务；
  - 完成/取消任务（原有逻辑保留）。

### 1.6 批量能力
- 已支持批量删除（含前后端）：
  - 订单中心、产品管理、采购单管理、部件库存。
- 规则遵守单删约束（例如订单仅待处理可删、采购单仅草稿可删等）。

## 2. 页面与交互状态

- 全站做过一轮样式重构（简约风格），包含响应式与列表分页（默认 10 条）。
- 转寄候选中已补全：
  - 当前挂靠与推荐来源的姓名/电话/地址/来源预定日期；
  - 当前挂靠与推荐来源的距离文案；
  - 推荐寄出日期（转寄来源：来源预定+1；仓库发货：目标预定-提前发货天数）。

## 3. 数据与测试状态

  - `python manage.py check`
  - `python manage.py test apps.core.tests -v 1`

## 4. 当前已知边界/注意事项

- 转寄中心“候选池”是调度视图，任务生成与推荐更新已分离，不要混用按钮语义。
- 若要对“来源筛选口径（是否按今天截断）”做业务调整，需要明确仅影响展示还是影响创建订单匹配。
- 历史数据如果存在旧逻辑残留，优先通过“重新推荐 + 生成任务”链路重建，减少脏挂靠影响。
