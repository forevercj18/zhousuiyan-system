## 6. 验收口径（Phase A Done Definition）

满足以下全部条件视为 Phase A 完成：
1. 动作级权限可用，关键动作无越权。
2. 审计日志可追踪关键字段变更前后值。
3. 页面与API统计字段口径一致。
4. 角色看板V1可按角色展示核心指标并可钻取。
5. 自动化回归通过，人工冒烟清单通过。
6. 提供可执行的发布与回滚方案。

---

## 7. 风险点与缓解

1. 权限改造影响面广：采用“默认兼容 + 渐进开关”方式。
2. 审计写入增加负担：关键动作异步化或批量写入。
3. 统计查询性能：先落地索引与缓存策略。
4. 前后端联调风险：先冻结接口契约再开发。
