# 项目阶段总结（2026-03-02）

分支：`main`  
范围：订单中心、转寄中心、库存/挂靠规则、系统设置、产品管理图片能力、全站样式重构

## 1. 本阶段已完成

### 1.1 订单与转寄主链路
- 订单中心与订单处理已合并为统一入口（订单中心）。
- 订单状态流转已稳定：`pending -> confirmed -> delivered -> returned -> completed`（含转寄分支）。
- “确认/发货”改为单弹窗输入（押金 + 可选运单号）。
- 押金默认值改为订单“应收押金总额”（明细押金*数量汇总），不再默认 0。
- “标记归还”录入回收单号逻辑已统一到 AppUI 弹窗。

### 1.2 转寄业务规则与闭环
- 创建订单时支持“自动匹配最优来源单”与“手工指定来源单/强制仓库发货”。
- 转寄候选排序规则已落地：
  - 第一排序：日期匹配度（以 `目标预定 + buffer_days` 为中心）
  - 第二排序：地址距离（本地省市坐标近似）
- 转寄任务与候选池分离：
  - `重新推荐`：仅更新挂靠
  - `生成任务`：基于当前挂靠创建任务
- 转寄任务“完成”闭环：
  - 单次录入一个运单号
  - 目标单变更为已发货并写入运单
  - 来源单标记为转寄完成路径（非回仓路径）
  - 库存不回仓释放
- 订单中心“转寄中心操作”拦截逻辑已收敛：
  - 仅在“仍作为有效来源挂靠”时拦截
  - 仓库发货且未被占用时可直接归还
- 订单列表新增“作为来源单占用中（N）”提示，解决“看起来仓库发货却被拦截”的认知问题。

### 1.3 库存与挂靠显示
- 可用库存展示按仓库实时口径输出，支持超占识别。
- 订单列表/工作台/详情页挂靠展示做了去重与合并展示修正。
- 修复过“同一来源单重复显示”问题。

### 1.4 前端交互与样式
- 全站完成一轮 Bootstrap 风格统一与响应式适配。
- 转寄中心拆分为 Tab 分栏（候选 / 任务），候选分页改为 5 条/页。
- 修复多个样式改造引发的事件失效问题（特别是订单创建页动态逻辑与弹窗层级）。
- 修复转寄中心与系统设置页 Tab 的 hover 黑底污染问题（全局 `.nav-item:hover` 冲突）。

### 1.5 系统设置重构
- 系统设置改为 Tab 分组：
  - `订单规则`
  - `转寄规则`
  - `仓库信息`
  - `系统与界面`
- 保存后保持当前 Tab，不再跳回默认页。

### 1.6 产品管理能力增强
- SKU 增加图片字段并完成迁移：`core.0003_sku_image`。
- 新建/编辑支持上传图片、清除图片。

## 2. 当前关键规则（已对齐）

1. 订单回流方式二选一  
- 回仓回流：走“标记归还”  
- 转寄回流：走“转寄任务完成”

2. 一旦订单被有效挂靠为来源单  
- 订单中心不允许直接“标记归还”  
- 必须走转寄中心闭环

3. 转寄任务完成后的库存  
- 不回仓释放，继续处于链路占用状态

4. 已发货及之后状态  
- 不允许编辑订单（前后端双重限制）
