### 阶段2.2：核心业务逻辑（优先级：P0）

#### 任务2.1：库存校验与排期计算
- **工作量**：3天
- **负责模块**：apps/core/services/inventory_service.py
- **交付物**：
  - check_sku_availability()：检查SKU在指定日期范围的可用性
  - calculate_occupation_period()：计算订单占用期
  - get_available_skus()：获取指定日期可用的SKU列表
- **业务规则**：
  - 占用期 = [发货日, 回收日]
  - 发货日 = 使用日期 - 发货提前天数
  - 回收日 = 使用日期 + 租赁天数 + 回收延后天数
  - 检查占用期内SKU库存是否充足
- **验收标准**：
  - 创建订单时自动校验库存
  - 库存不足时返回明确错误信息
  - 排期看板数据准确

#### 任务2.2：订单状态流转
- **工作量**：2天
- **负责模块**：apps/core/services/order_service.py
- **交付物**：
  - 订单状态机：待确认→待打包→待发货→已发货→待回收→已完成
  - 状态转换方法：mark_packed(), mark_shipped(), mark_returned()
  - 状态转换权限校验
  - 状态转换时自动更新库存占用
- **验收标准**：
  - 工作台操作按钮正常工作
  - 状态转换记录操作日志
  - 非法状态转换被拒绝

#### 任务2.3：转寄匹配算法
- **工作量**：3天
- **负责模块**：apps/core/services/transfer_service.py
- **交付物**：
  - find_transfer_candidates()：查找可转寄的订单对
  - calculate_transfer_benefit()：计算转寄节省成本
  - create_transfer_task()：创建转寄任务
  - execute_transfer()：执行转寄（更新订单发货地址）
- **匹配规则**：
  - 订单A回收日期 < 订单B发货日期
  - 间隔天数 <= 最大转寄间隔天数
  - 同一SKU
  - 订单A状态=已发货，订单B状态=待打包/待发货
  - 订单B回收地址 = 订单A发货地址（或距离近）
- **验收标准**：
  - 自动生成转寄候选列表
  - 手动确认后创建转寄任务
  - 转寄任务执行后更新订单信息

#### 任务2.4：采购与部件库存管理
- **工作量**：2天
- **负责模块**：apps/core/services/procurement_service.py
- **交付物**：
  - create_purchase_order()：创建采购单
  - mark_arrived()：标记到货
  - mark_stocked()：标记入库（更新部件库存）
  - check_low_stock_parts()：检查低库存部件
  - record_movement()：记录出入库流水
- **业务规则**：
  - 采购单状态：待下单→已下单→已到货→已入库
  - 标记入库时自动增加部件库存
  - 部件库存 < 安全库存时预警
- **验收标准**：

```python
from django.contrib.auth.models import User

    class Meta:

        verbose_name = '用户资料'
        verbose_name_plural = '用户资料'
```

```python
    CATEGORY_CHOICES = [
        ('theme_package', '主题套餐'),
        ('single_item', '单品'),
    ]
    
    code = models.CharField(max_length=50, unique=True, verbose_name='SKU编码')
    name = models.CharField(max_length=200, verbose_name='名称')
    category = models.CharField(max_length=50, choices=CATEGORY_CHOICES, verbose_name='分类')
    rental_price = models.DecimalField(max_digits=10, decimal_places=2, verbose_name='租金')
    deposit = models.DecimalField(max_digits=10, decimal_places=2, verbose_name='押金')
    stock = models.IntegerField(default=0, verbose_name='库存数量')
    description = models.TextField(blank=True, verbose_name='描述')
    is_active = models.BooleanField(default=True, verbose_name='是否启用')
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)
    
    class Meta:
        db_table = 'skus'
        verbose_name = 'SKU'
        verbose_name_plural = 'SKU'
        indexes = [
            models.Index(fields=['code']),
            models.Index(fields=['is_active']),
        ]
    
    def __str__(self):
        return f"{self.code} - {self.name}"
```

```python
class Part(models.Model):
    """部件/备件"""

        ('backdrop', '背景板'),
        ('decoration', '装饰品'),
        ('balloon', '气球'),
        ('prop', '道具'),

    code = models.CharField(max_length=50, unique=True, verbose_name='部件编码')

    specification = models.CharField(max_length=200, blank=True, verbose_name='规格')
    unit = models.CharField(max_length=20, default='个', verbose_name='单位')
    stock = models.IntegerField(default=0, verbose_name='当前库存')
    min_stock = models.IntegerField(default=0, verbose_name='安全库存')
    unit_cost = models.DecimalField(max_digits=10, decimal_places=2, default=0, verbose_name='单位成本')
    location = models.CharField(max_length=100, blank=True, verbose_name='存放位置')
    is_active = models.BooleanField(default=True, verbose_name='是否启用')
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)
    
    class Meta:
        db_table = 'parts'
        verbose_name = '部件'
        verbose_name_plural = '部件'
        indexes = [
            models.Index(fields=['code']),
            models.Index(fields=['category']),
        ]
    
    def __str__(self):
        return f"{self.code} - {self.name}"
    
    @property
    def is_low_stock(self):
        """是否低库存"""
        return self.stock < self.min_stock
```

### 2.5 订单模型（Order）

```python
class Order(models.Model):
    """租赁订单"""
    STATUS_CHOICES = [
        ('pending', '待确认'),
        ('confirmed', '已确认'),
        ('to_pack', '待打包'),
        ('to_ship', '待发货'),
        ('shipped', '已发货'),
        ('to_return', '待回收'),
        ('returned', '已回收'),
        ('completed', '已完成'),
        ('cancelled', '已取消'),
    ]
    
    order_no = models.CharField(max_length=50, unique=True, verbose_name='订单号')
    sku = models.ForeignKey(SKU, on_delete=models.PROTECT, verbose_name='SKU')
    
    # 客户信息
    customer_name = models.CharField(max_length=100, verbose_name='客户姓名')
    customer_phone = models.CharField(max_length=20, verbose_name='客户电话')
    shipping_address = models.TextField(verbose_name='发货地址')
    return_address = models.TextField(verbose_name='回收地址')
    
    # 租赁信息
    event_date = models.DateField(verbose_name='使用日期')
    rental_days = models.IntegerField(default=3, verbose_name='租赁天数')
    rental_price = models.DecimalField(max_digits=10, decimal_places=2, verbose_name='租金')
    deposit = models.DecimalField(max_digits=10, decimal_places=2, verbose_name='押金')
    
    # 物流信息
    ship_date = models.DateField(verbose_name='发货日期')
    return_date = models.DateField(verbose_name='回收日期')
    tracking_no = models.CharField(max_length=100, blank=True, verbose_name='物流单号')
```

### 3.1 库存校验与排期计算

#### 业务规则
```python
# apps/core/services/inventory_service.py

from datetime import timedelta
from django.db.models import Q, Count
from apps.core.models import Order, SKU, SystemSettings

class InventoryService:
    """库存与排期服务"""
    
    @staticmethod
    def calculate_occupation_period(event_date, rental_days):
        """
        计算订单占用期
        
        Args:
            event_date: 使用日期
            rental_days: 租赁天数
            
        Returns:
            (ship_date, return_date): 发货日期和回收日期
        """
        settings = SystemSettings.get_settings()
        
        # 发货日 = 使用日期 - 发货提前天数
        ship_date = event_date - timedelta(days=settings.ship_advance_days)
        
        # 回收日 = 使用日期 + 租赁天数 + 回收延后天数
        return_date = event_date + timedelta(days=rental_days) + timedelta(days=settings.return_delay_days)
        
        return ship_date, return_date
    
    @staticmethod
    def check_sku_availability(sku_id, ship_date, return_date, exclude_order_id=None):
        """
        检查SKU在指定日期范围的可用性
        
        Args:
            sku_id: SKU ID
            ship_date: 发货日期
            return_date: 回收日期
            exclude_order_id: 排除的订单ID（编辑订单时使用）
            
        Returns:
            {
                'available': True/False,
                'total_stock': 总库存,
                'occupied': 占用数量,
                'available_count': 可用数量,
                'conflicting_orders': 冲突订单列表
            }
        """
        sku = SKU.objects.get(id=sku_id)
        
        # 查询占用期有重叠的订单
        # 重叠条件：订单的[ship_date, return_date]与查询的[ship_date, return_date]有交集
        # 即：订单.return_date >= 查询.ship_date AND 订单.ship_date <= 查询.return_date
        conflicting_orders = Order.objects.filter(
            sku_id=sku_id,
            status__in=['confirmed', 'to_pack', 'to_ship', 'shipped', 'to_return'],
            return_date__gte=ship_date,
            ship_date__lte=return_date
        )
        
        if exclude_order_id:
            conflicting_orders = conflicting_orders.exclude(id=exclude_order_id)
        
        occupied = conflicting_orders.count()
        available_count = sku.stock - occupied
        
        return {
            'available': available_count > 0,
            'total_stock': sku.stock,
            'occupied': occupied,
            'available_count': available_count,
            'conflicting_orders': list(conflicting_orders.values('order_no', 'customer_name', 'event_date'))
        }
    
    @staticmethod
    def get_calendar_matrix(start_date, end_date):
        """
        获取排期看板矩阵数据
        
        Args:
            start_date: 开始日期
            end_date: 结束日期
            
        Returns:
            {
                'dates': [日期列表],
                'skus': [SKU列表],
                'matrix': {
                    'sku_id': {
                        'date': {
                            'total': 总库存,
                            'occupied': 占用数量,
                            'available': 可用数量,
                            'orders': [订单列表]
                        }
                    }
                }
            }
        """
        from datetime import timedelta
        
        # 生成日期列表
        dates = []
        current_date = start_date
        while current_date <= end_date:
            dates.append(current_date)
            current_date += timedelta(days=1)
        
        # 获取所有活跃的SKU
        skus = SKU.objects.filter(is_active=True)
        
        # 获取日期范围内的所有订单
        orders = Order.objects.filter(
            status__in=['confirmed', 'to_pack', 'to_ship', 'shipped', 'to_return'],
            return_date__gte=start_date,
            ship_date__lte=end_date
        ).select_related('sku')
        
        # 构建矩阵
        matrix = {}
        for sku in skus:
            matrix[sku.id] = {}
            for date in dates:
                # 查找该日期占用该SKU的订单
                occupied_orders = [
                    order for order in orders
                    if order.sku_id == sku.id and order.ship_date <= date <= order.return_date
                ]
                
                occupied = len(occupied_orders)
                available = sku.stock - occupied
                
                matrix[sku.id][date.isoformat()] = {
                    'total': sku.stock,
                    'occupied': occupied,
                    'available': available,
                    'orders': [
                        {
                            'order_no': o.order_no,
                            'customer_name': o.customer_name,
                            'event_date': o.event_date.isoformat()
                        }
                        for o in occupied_orders
                    ]
                }
        
        return {
            'dates': [d.isoformat() for d in dates],
            'skus': [{'id': s.id, 'code': s.code, 'name': s.name, 'stock': s.stock} for s in skus],
            'matrix': matrix
        }
```

### 3.2 订单状态流转

#### 状态机设计
```
待确认(pending) → 已确认(confirmed) → 待打包(to_pack) → 待发货(to_ship) 
→ 已发货(shipped) → 待回收(to_return) → 已回收(returned) → 已完成(completed)
                                                                    ↓
                                                              已取消(cancelled)
```

#### 业务逻辑
```python
# apps/core/services/order_service.py

from django.db import transaction
from apps.core.models import Order, AuditLog

class OrderService:
    """订单服务"""
    
    # 允许的状态转换
    ALLOWED_TRANSITIONS = {
        'pending': ['confirmed', 'cancelled'],
        'confirmed': ['to_pack', 'cancelled'],
        'to_pack': ['to_ship', 'cancelled'],
        'to_ship': ['shipped', 'cancelled'],
        'shipped': ['to_return'],
        'to_return': ['returned'],
        'returned': ['completed'],
    }
    
    @staticmethod
    @transaction.atomic
    def create_order(data, user):
        """
        创建订单
        
        Args:
            data: 订单数据
            user: 创建人
            
        Returns:
            Order对象
            
        Raises:
            ValueError: 库存不足
        """
        from apps.core.services.inventory_service import InventoryService
        
        # 计算占用期
        ship_date, return_date = InventoryService.calculate_occupation_period(
            data['event_date'],
            data['rental_days']
        )
        
        # 检查库存
        availability = InventoryService.check_sku_availability(
            data['sku_id'],
            ship_date,
            return_date
        )
        
        if not availability['available']:
            raise ValueError(
                f"SKU库存不足。总库存：{availability['total_stock']}，"
                f"已占用：{availability['occupied']}，可用：{availability['available_count']}"
            )
        
        # 创建订单
        order = Order.objects.create(
            sku_id=data['sku_id'],
            customer_name=data['customer_name'],
            customer_phone=data['customer_phone'],
            shipping_address=data['shipping_address'],
            return_address=data['return_address'],
            event_date=data['event_date'],
            rental_days=data['rental_days'],
            rental_price=data['rental_price'],
            deposit=data['deposit'],
            ship_date=ship_date,
            return_date=return_date,
            tracking_no=data.get('tracking_no', ''),
            notes=data.get('notes', ''),
            status='confirmed',
            created_by=user
        )
        
        # 记录日志
        AuditLog.objects.create(
            user=user,
            action='create',
            module='order',
            description=f"创建订单：{order.order_no}，客户：{order.customer_name}"
        )
        
        return order
    
    @staticmethod
    @transaction.atomic
    def change_status(order_id, new_status, user, notes=''):
        """
        变更订单状态
        
        Args:
            order_id: 订单ID
            new_status: 新状态
            user: 操作人
            notes: 备注
            
        Raises:
            ValueError: 非法状态转换
        """
        order = Order.objects.select_for_update().get(id=order_id)
        
        # 检查状态转换是否合法
        allowed = OrderService.ALLOWED_TRANSITIONS.get(order.status, [])
        if new_status not in allowed:
            raise ValueError(
                f"不允许从 {order.get_status_display()} 转换到 {dict(Order.STATUS_CHOICES)[new_status]}"
            )
        
        old_status = order.status
        order.status = new_status
        if notes:
            order.notes = f"{order.notes}\n{notes}" if order.notes else notes
        order.save()
        
        # 记录日志
        AuditLog.objects.create(
            user=user,
            action='status_change',
            module='order',
            description=f"订单 {order.order_no} 状态从 {dict(Order.STATUS_CHOICES)[old_status]} 变更为 {order.get_status_display()}"
        )
        
        return order
    
    @staticmethod
    def mark_packed(order_id, user):
        """标记已打包"""
        return OrderService.change_status(order_id, 'to_ship', user, '已打包')
    
    @staticmethod
    def mark_shipped(order_id, user, tracking_no=''):
        """标记已发货"""
        order = Order.objects.get(id=order_id)
        if tracking_no:
            order.tracking_no = tracking_no
            order.save()
        return OrderService.change_status(order_id, 'shipped', user, f'已发货，单号：{tracking_no}')
    
    @staticmethod
    def mark_returned(order_id, user):
        """标记已回收"""
        return OrderService.change_status(order_id, 'returned', user, '已回收')
```

### 3.3 转寄匹配算法

#### 匹配规则
```python
# apps/core/services/transfer_service.py

from datetime import timedelta
from django.db.models import Q
from apps.core.models import Order, Transfer, SystemSettings, AuditLog

class TransferService:
    """转寄服务"""
    
    @staticmethod
    def find_transfer_candidates():
        """
        查找可转寄的订单对
        
        Returns:
            [
                {
                    'source_order': 源订单对象,
                    'target_order': 目标订单对象,
                    'interval_days': 间隔天数,
                    'estimated_savings': 预计节省成本
                }
            ]
        """
        settings = SystemSettings.get_settings()
        candidates = []
        
        # 查找已发货的订单（作为源订单）
        source_orders = Order.objects.filter(
            status='shipped',
            is_transfer_source=False
        ).select_related('sku')
        
        for source in source_orders:
            # 查找可能的目标订单
            # 条件：
            # 1. 同一SKU
            # 2. 状态为待打包或待发货
            # 3. 目标订单发货日期 > 源订单回收日期
            # 4. 间隔天数 <= 最大转寄间隔天数
            # 5. 目标订单回收地址 = 源订单发货地址（简化处理）
            
            max_ship_date = source.return_date + timedelta(days=settings.max_transfer_interval)
            
            target_orders = Order.objects.filter(
                sku=source.sku,
                status__in=['to_pack', 'to_ship'],
                is_transfer_target=False,
                ship_date__gt=source.return_date,
                ship_date__lte=max_ship_date,
                return_address=source.shipping_address  # 简化：地址完全匹配
            )
            
            for target in target_orders:
                interval_days = (target.ship_date - source.return_date).days
                
                # 计算节省成本（假设每次物流成本50元）
                estimated_savings = 50.0  # 简化计算
                
                candidates.append({
                    'source_order': source,
                    'target_order': target,
                    'interval_days': interval_days,
                    'estimated_savings': estimated_savings
                })
        
        # 按间隔天数排序（间隔越小越优先）
        candidates.sort(key=lambda x: x['interval_days'])
        
        return candidates
    
    @staticmethod
    @transaction.atomic
    def create_transfer_task(source_order_id, target_order_id, user):
        """
        创建转寄任务
        
        Args:
            source_order_id: 源订单ID
            target_order_id: 目标订单ID
            user: 创建人
            
        Returns:
            Transfer对象
        """
        source = Order.objects.get(id=source_order_id)
        target = Order.objects.get(id=target_order_id)
        
        # 验证
        if source.sku_id != target.sku_id:
            raise ValueError("源订单和目标订单的SKU不一致")
        
        if target.ship_date <= source.return_date:
            raise ValueError("目标订单发货日期必须晚于源订单回收日期")
        
        interval_days = (target.ship_date - source.return_date).days
        estimated_savings = 50.0
        
        # 创建转寄任务
        transfer = Transfer.objects.create(
            source_order=source,
            target_order=target,
            interval_days=interval_days,
            estimated_savings=estimated_savings,
            status='pending',
            created_by=user
        )
        
        # 记录日志
        AuditLog.objects.create(
            user=user,
            action='create',
            module='transfer',
            description=f"创建转寄任务：{transfer.task_no}，源订单：{source.order_no}，目标订单：{target.order_no}"
        )
        
        return transfer
    
    @staticmethod
    @transaction.atomic
    def execute_transfer(transfer_id, user):
        """
        执行转寄任务
        
        Args:
            transfer_id: 转寄任务ID
            user: 执行人
        """
        from django.utils import timezone
        
        transfer = Transfer.objects.select_for_update().get(id=transfer_id)
        
        if transfer.status != 'pending':
            raise ValueError("只能执行待执行状态的转寄任务")
        
        # 更新目标订单的发货地址为源订单的回收地址
        target = transfer.target_order
        target.shipping_address = transfer.source_order.return_address
        target.save()
        
        # 标记订单
        transfer.source_order.is_transfer_source = True
        transfer.source_order.save()
        
        transfer.target_order.is_transfer_target = True
        transfer.target_order.save()
        
        # 更新转寄任务状态
        transfer.status = 'executed'
        transfer.executed_at = timezone.now()
        transfer.executed_by = user
        transfer.save()
        
        # 记录日志
        AuditLog.objects.create(
            user=user,
            action='update',
            module='transfer',
            description=f"执行转寄任务：{transfer.task_no}"
        )
        
        return transfer
```

```python
# apps/core/decorators.py

from functools import wraps
from django.http import HttpResponseForbidden
from django.shortcuts import redirect

def role_required(*allowed_roles):
    """
    角色权限装饰器
    
    用法：
    @role_required('super_admin', 'business_manager')
    def my_view(request):
        ...
    """
    def decorator(view_func):
        @wraps(view_func)
        def wrapper(request, *args, **kwargs):
            if not request.user.is_authenticated:
                return redirect('login')
            
            user_role = request.user.profile.role
            if user_role not in allowed_roles:
                return HttpResponseForbidden("您没有权限访问此页面")
            
            return view_func(request, *args, **kwargs)
        return wrapper
    return decorator


def permission_required(permission_name):
    """
    权限装饰器（基于权限名称）
    
    用法：
    @permission_required('can_edit_order')
    def my_view(request):
        ...
    """
    PERMISSIONS = {
        'can_edit_order': ['super_admin', 'business_manager', 'customer_service'],
        'can_delete_order': ['super_admin', 'business_manager'],
        'can_manage_sku': ['super_admin', 'warehouse_manager'],
        'can_manage_procurement': ['super_admin', 'warehouse_manager', 'warehouse_operator'],
        'can_manage_users': ['super_admin'],
        'can_view_audit_logs': ['super_admin'],
    }
    
    def decorator(view_func):
        @wraps(view_func)
        def wrapper(request, *args, **kwargs):
            if not request.user.is_authenticated:
                return redirect('login')
            
            user_role = request.user.profile.role
            allowed_roles = PERMISSIONS.get(permission_name, [])
            
            if user_role not in allowed_roles:
                return HttpResponseForbidden("您没有权限执行此操作")
            
            return view_func(request, *args, **kwargs)
        return wrapper
    return decorator
```

```python
# apps/core/middleware.py

from apps.core.models import AuditLog

class AuditLogMiddleware:
    """操作日志中间件"""
    
    def __init__(self, get_response):
        self.get_response = get_response
    
    def __call__(self, request):
        response = self.get_response(request)
        
        # 只记录已登录用户的POST/PUT/DELETE请求
        if request.user.is_authenticated and request.method in ['POST', 'PUT', 'DELETE']:
            # 排除某些路径
            if not request.path.startswith('/api/auth/') and not request.path.startswith('/static/'):
                self._log_action(request, response)
        
        return response
    
    def _log_action(self, request, response):
        """记录操作日志"""
        # 根据路径判断模块和操作类型
        path = request.path
        method = request.method
        
        module_map = {
            '/orders': 'order',
            '/skus': 'sku',
            '/procurement': 'purchase',
            '/transfers': 'transfer',
            '/users': 'user',
            '/settings': 'settings',
        }
        
        action_map = {
            'POST': 'create',
            'PUT': 'update',
            'DELETE': 'delete',
        }
        
        module = 'other'
        for prefix, mod in module_map.items():
            if path.startswith(prefix):
                module = mod
                break
        
        action = action_map.get(method, 'update')
        
        # 获取IP地址
        x_forwarded_for = request.META.get('HTTP_X_FORWARDED_FOR')
        if x_forwarded_for:
            ip_address = x_forwarded_for.split(',')[0]
        else:
            ip_address = request.META.get('REMOTE_ADDR')
        
        # 创建日志
        try:
            AuditLog.objects.create(
                user=request.user,
                action=action,
                module=module,
                description=f"{action_map.get(method, '操作')} {path}",
                ip_address=ip_address,
                user_agent=request.META.get('HTTP_USER_AGENT', '')[:500]
            )
        except Exception as e:
            # 日志记录失败不应影响正常请求
            pass
```

```javascript
const API = {
    baseURL: '/api',
    
    // 通用请求方法
    async request(method, url, data = null) {
        const options = {
            method: method,
            headers: {
                'Content-Type': 'application/json',
                'X-CSRFToken': this.getCookie('csrftoken')
            }
        };
        
        if (data) {

        try {
            const response = await fetch(this.baseURL + url, options);
            
            if (!response.ok) {
                const error = await response.json();

            }
```

## 8. 技术难点分析

### 8.1 库存校验与排期计算

#### 难点描述
- 需要准确计算订单占用期（考虑发货提前天数、回收延后天数）
- 判断日期范围重叠的算法
- 排期看板需要高效查询大量订单数据

#### 解决方案
1. **日期重叠判断**：
   - 两个日期区间 [A1, A2] 和 [B1, B2] 重叠的条件：`A2 >= B1 AND A1 <= B2`
   - 使用数据库索引优化查询性能

2. **排期看板优化**：
   - 使用 `select_related` 和 `prefetch_related` 减少数据库查询次数
   - 对于矩阵模式，一次性查询所有相关订单，在内存中计算
   - 使用Redis缓存排期数据（缓存5分钟）

3. **并发控制**：
   - 创建订单时使用数据库事务
   - 使用 `select_for_update()` 锁定SKU记录，防止超卖

```python
        """测试库存检查（无冲突）"""
        result = InventoryService.check_sku_availability(
            self.sku.id,
            date(2026, 3, 1),
            date(2026, 3, 5)
        )
        
        self.assertTrue(result['available'])
        self.assertEqual(result['total_stock'], 5)
        self.assertEqual(result['occupied'], 0)
        self.assertEqual(result['available_count'], 5)
    
    def test_check_sku_availability_with_conflict(self):
        """测试库存检查（有冲突）"""
        # 创建5个占用该SKU的订单
        for i in range(5):
            Order.objects.create(
                sku=self.sku,

                return_date=date(2026, 3, 20),

            )
        
        # 检查同一时间段的库存
        result = InventoryService.check_sku_availability(
```

```python
                'rental_price': sku_data['rental_price'],
                'deposit': sku_data['deposit'],
                'stock': sku_data['stock'],
                'description': sku_data['description'],

        )

def migrate_parts():
    """迁移部件数据"""
```
