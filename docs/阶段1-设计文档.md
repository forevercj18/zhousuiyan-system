## 3) 每个页面的UI元素清单

### 3.1 登录页 (`login.html`)

## 4) Mock数据策略

### 数据来源
- 统一存放在 `apps/core/mock_data.py`
- 使用Python字典/列表结构
- 数据结构与未来Model字段对齐

### Mock数据清单
- **MOCK_SKUS**：5条租赁套装SKU
- **MOCK_ORDERS**：20条订单
- **MOCK_TRANSFER_CANDIDATES**：5条转寄候选
- **MOCK_TRANSFER_TASKS**：3条转寄任务
- **MOCK_AUDIT_LOGS**：20条操作日志
- **MOCK_PURCHASE_ORDERS**：10条采购单
- **MOCK_PURCHASE_ORDER_ITEMS**：若干明细行
- **MOCK_PARTS**：30条部件库存
- **MOCK_PARTS_MOVEMENTS**：30条出入库流水
- **MOCK_SETTINGS**：系统设置参数
- **MOCK_DASHBOARD_STATS**：仪表盘统计数据
- **MOCK_CALENDAR_DATA**：排期看板数据
- **MOCK_USERS**：6条用户数据（含5种角色）**【新增 - 2026-02-27】**

### 模板读取方式
- 视图函数从 `mock_data.py` 导入数据
- 通过 `context` 传递给模板
- 模板使用 Django 模板语法渲染

### 第二阶段替换策略
- 视图函数改为从 Model.objects.all() 查询
- context 数据结构保持不变
- 模板无需修改

## 5) Windows傻瓜式部署方案

### 5.1 Docker配置

**Dockerfile**
- 基础镜像：Python 3.11-slim
- 安装依赖：requirements.txt
- 收集静态文件：collectstatic
- 启动服务：Gunicorn（2 workers）
- 暴露端口：8000

**docker-compose.yml**
- 服务：web
- 端口映射：8000:8000
- 数据库挂载：db.sqlite3
- 启动命令：migrate + 创建管理员 + gunicorn

### 5.2 初始化管理员账号

**scripts/create_superuser.py**
- 默认账号：admin / admin123
- 自动检测是否已存在
- 容器启动时自动执行

### 5.3 部署步骤（README.md）

1. 安装 Docker Desktop for Windows
2. 下载项目到本地
3. 在项目目录执行：`docker compose up -d --build`
4. 访问 http://localhost:8000
5. 使用默认账号登录：admin / admin123

### 5.4 静态文件托管方案
- 使用 **whitenoise** 中间件
- 配置在 `settings.py`
- 容器启动时执行 `collectstatic`

### 6.2 第一阶段实现
- 所有API路由已定义
